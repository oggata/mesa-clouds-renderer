## 本書について

本書は、Hugging FaceのLeRobotプロジェクトが公開したオープンソースのロボットアーム **SO-101** を題材に、ロボットを組み立てるところから、模倣学習・強化学習でタスクを覚えさせ、シミュレーションと実機をつなぐところまでを、実際に手を動かしながら学ぶ実践ガイドです。

数百万円の産業用ロボットと研究室が必要だった模倣学習が、部品代**約5万円**のアームと無料のソフトウェアで、個人の机の上で試せる時代になりました。本書は、その最短ルートを一冊にまとめたものです。

## 対象読者

- ロボットアームで模倣学習・強化学習を実際に動かしてみたい方
- Physical AI（物理世界で行動するAI）に興味があり、手元で検証したい方
- LeRobotエコシステム（SO-101、Hugging Face Hub、Isaac Sim、GR00T）を体系的に把握したい方

Pythonの基本と、ターミナルでのコマンド操作に抵抗がなければ読み進められます。機械学習やロボティクスの専門知識は前提としません。必要な概念はその都度説明します。

# 第1章　SO-101とは — オープンソースロボティクスの民主化

## 1.1 ロボティクス研究の転換点

2025年5月、Hugging Faceが主導するLeRobotプロジェクトから、オープンソースのロボットアーム **SO-101（Standard Open-101 Arms）** が正式に公開されました。これは単なる新型アームの登場ではなく、**「ロボティクス × AI」の実験が、研究室レベルから個人の机の上へと降りてきた**歴史的な転換点です。

従来、模倣学習の研究には数百万円規模の産業用ロボットか、大学・企業の研究室が必要でした。SO-101は部品代**約5万円**で構築でき、設計図も組み立て方法も完全に公開されています。学生、個人研究者、ホビイストまで、誰もが最先端のPhysical AI研究に参加できるようになったのです。

![画像](https://static.zenn.studio/user-upload/e03b90ba6aa9-20251221.gif)

同種のオープンソースロボットは OpenDuckMini や Reachy Mini なども登場しており、それらを動かすデータも急増しています。Hugging Faceの公開データセットは60万件以上に成長し、なかでもロボティクスは最も急成長しているセグメントです（2024年の1,145件から2025年には26,991件へ、わずか3年でカテゴリランキング44位から1位へ）。

[from the bottom to the top: robotics datasets lead on Hugging Face](https://aiworld.eu/story/from-the-bottom-to-the-top-robotics-datasets-lead-on-hugging-face)

![画像](https://static.zenn.studio/user-upload/f96d75dd8495-20251225.png)

そして重要なのは、こうした安価なアームで学習した成果が、Franka Research 3、ALOHA（両腕）、その他Open X-Embodimentに含まれる産業グレードのロボットでも活用できる点です。同一のツールチェーンを使うため、ソフトウェアを書き換える必要がありません。

## 1.2 LeRobotプロジェクトとは

LeRobotは、Hugging Faceが2024年から本格始動させたオープンソースのロボティクス教育・研究プロジェクトです。自然言語処理を「Transformers」ライブラリで民主化したHugging Faceが、同じ理念をロボティクスに持ち込みました。核心は3つの柱にあります。

- **ハードウェアのオープン化**：SO-100、SO-101など、完全にオープンソースな設計を公開
- **ソフトウェアのオープン化**：データ収集から学習・評価までの一貫したパイプラインを提供
- **データのオープン化**：Hugging Face Hubを通じた学習データセットの共有プラットフォーム

この3つが揃うことで、世界中の研究者が同じ基盤の上で実験し、結果を比較し、知見を共有できる——自然言語処理でHugging Faceが成し遂げたのと同じ「民主化」がロボティクスでも起きています。

:::message
📷 **画像プレースホルダー**：LeRobotの3つの柱（ハードウェア/ソフトウェア/データのオープン化）が Hugging Face Hub を中心に連携する概念図。「設計図の共有→パイプライン→データセット共有→再現」の循環を矢印で示す。
:::

公式リポジトリ：[huggingface/lerobot](https://github.com/huggingface/lerobot)

### Hugging Face Hub / NVIDIA Isaac GR00Tとの連携

LeRobotの大きな特徴は、Hugging Face Hub や NVIDIA Isaac GR00T との統合です。収集した動作データは `push_to_hub` 一つでアップロードでき、学習済みポリシーもバージョン管理しながら公開でき、誰でも同じデータ・同じモデルで実験を再現できます。

たとえば「散らかったデスクを片付ける」タスクのデータセット（[GR00T N1.5 for LeRobot SO-101](https://huggingface.co/blog/nvidia/gr00t-n1-5-so101-tuning)）は、世界中の誰もがダウンロードして自分のSO-101で実行できます。こうしたプラットフォームは、複数の異なるロボットにまたがってスキルを転移できる**クロスエンボディメント**なモデルの実現にもつながります。「うちのロボットでは動いた」という再現性のない研究から、「このデータ・このモデル・この環境で成功率95%」という科学的な研究への進化です。

## 1.3 SO-101の誕生背景

SO-101は、The Robot Studioが開発した前身 SO-100 の改良版で、LeRobotの「定番」ハードウェアです。SO-100からは、各ジョイントのギア比の最適化（リーダーとフォロワーで異なるモーター仕様）、3Dプリント部品の設計改善による構造強度の向上、配線の整理、そしてLeRobotライブラリとのネイティブ統合、といった点が改善されました。

名称の「Standard Open」には「これが標準になる」という意図が込められています——ロボット研究のベンチマークとなる**標準**プラットフォーム、設計・ソフト・データすべてが**オープン**、そして誰もが最初に手に取る「ロボティクス**101**」の教材。2025年6月のワールドワイドハッカソンでは、多くの参加者がSO-101で物体の分類・配置、複雑な組み立て、双腕協調まで、わずか数週間で実現しました。

## 1.4 従来のロボットアームとの違い

| 特徴 | 産業用ロボットアーム | SO-101 |
|---|---|---|
| 価格 | 300万円〜数千万円 | 約4〜5万円（部品代） |
| 精度 | 反復精度 ±0.05mm | 反復精度 ±1〜2mm |
| ペイロード | 3kg〜数百kg | 約500g（Pro版で1kg程度） |
| 制御方式 | 専用コントローラー | USB経由でPC制御 |
| プログラミング | 専用言語（RAPID等） | Python + LeRobot |
| 学習データ | 非公開 | Hugging Face Hubで共有 |
| カスタマイズ性 | 限定的 | 完全にオープン |

産業用ロボットが「高精度・高信頼性・クローズド」なのに対し、SO-101は「低コスト・学習可能・オープン」という対極の設計思想を持ちます。産業用を置き換えるものではなく、**研究と教育に特化した新しいカテゴリー**——模倣学習の実験、アルゴリズムの公平なベンチマーク、VLAなどPhysical AIの検証、手頃な教育ツールとしての価値がここにあります。

特に重要なのが**リーダーアームとフォロワーアームの2台構成**です。人間がリーダーアームを動かすとフォロワーアームが追従する「テレオペレーション」により、直感的にデータを収集できます。この仕組みが、専門家しか扱えなかった模倣学習を、誰でも実践できる技術に変えました。

## 1.5 模倣学習の基礎概念

**模倣学習（Imitation Learning）** は、人間のデモンストレーションからロボットが行動を学ぶ手法です。従来のロボットプログラミングでは、すべての動作を明示的に記述する必要がありました。

```python
move_to(x=0.3, y=0.2, z=0.15)   # 位置1に移動
close_gripper()                  # グリッパーを閉じる
move_to(x=0.4, y=-0.1, z=0.15)  # 位置2に移動
```

模倣学習は、これを根本から変えます。人間がリーダーアームでタスクを実演し、カメラ画像と関節角度を記録し、ニューラルネットワークが「この画像を見たらどう動くべきか」を学習し、学習済みモデルでロボットが自律的にタスクを実行する——**「どう動くか」をプログラムする代わりに、「こう動いてほしい」を見せるだけ**という点が革新的です。

SO-101での流れは、①リーダーとフォロワーを並べて配置し、②作業領域を撮るカメラを設置し、③人間がリーダーを操作してデータ（カメラ画像＝観測、関節角度＝状態、次の目標角度＝アクション）を同時記録し、④同じタスクを30〜100回繰り返して多様なデータを集め、⑤ニューラルネットワークを訓練し、⑥学習済みモデルで自律実行する、というものです。この一連の流れが、LeRobotによって**数行のコマンド**で実行できます。

:::message
📷 **画像プレースホルダー**：模倣学習ワークフローの全体図。「テレオペでデモ収集（観測＋状態＋アクション）→ データセット化 → ニューラルネット学習 → 自律実行」の1周を、SO-101のリーダー/フォロワーとカメラのイラストとともに示す。本書全体の地図になる図。
:::

## 1.6 模倣学習のアプローチ

SO-101で最もよく使われるのが **ACT（Action Chunking with Transformers）** です。2023年にスタンフォード大学が発表し、低コストなハードウェアでも高精度な模倣学習が可能なことを示しました。核心は2つ——**Conditional VAE**で現在の観測から「未来の一連の行動（アクションチャンク）」をまとめて予測すること、そして**Temporal Ensembling**で複数時刻の予測を加重平均し滑らかで安定した動作を得ることです。一歩先だけでなく数秒先まで見越して動けるため、人間らしい滑らかな動作になります（詳細は第11章）。

![画像](https://static.zenn.studio/user-upload/4c25ed55ab2f-20260106.gif)

[ACT解説動画（YouTube）](https://www.youtube.com/watch?v=J1sv_WyPRzY)

LeRobotはACT以外にも、拡散モデルで行動を生成する **Diffusion Policy**、Model-Based強化学習の **TDMPC**、言語指示も理解する次世代の **VLA（Vision-Language-Action Models）**（Physical Intelligence社の π0、コンパクトな SmolVLA など）をサポートします。本書は主にACTを軸に解説し、これら先進的なアプローチは第12章・第18章で扱います。

## 1.7 Physical AIとの関連

SO-101は、より大きな文脈である **Physical AI**——物理世界を理解し、物理世界で行動できるAI——の実験プラットフォームです。デジタル空間の知的処理に特化した従来のAIと異なり、Physical AIは知覚（カメラで観測）・理解（3次元空間や物理法則の理解）・計画（行動計画）・制御（精密なロボット制御）・学習（経験からの改善）を統合的に扱います。SO-101 + LeRobot は、その全要素を一つの環境で扱えます。

特に注目すべきは、**カメラ画像から直接モーター制御まで一つのニューラルネットワークで処理する End-to-End 学習**です。

```
モジュラー：  カメラ → 物体検出 → 位置推定 → 軌道計画 → モーター制御（各段階が別アルゴリズム）
End-to-End： カメラ画像 → [ニューラルネットワーク] → モーター角度（一つのモデルで直接変換）
```

このアプローチは、Teslaの自動運転で実証されているように、大量のデータがあれば従来手法を超える可能性があります。SO-101は、これをマニピュレーション（物体操作）の領域で検証する理想的なプラットフォームなのです。

## ステップ0：準備するもの

「はじめに」で挙げた通り、SO-101キット（リーダー＋フォロワー）、USBカメラ、macOS + Conda環境のPC、Hugging Faceアカウント、そして学習用のクラウドGPU（Google Colab等）を用意します。

## ステップ1〜3：アームを「手で動かせる」状態にする

まずソフトを整え（第5章）、モーターIDの設定とキャリブレーションを行い（第6章）、リーダーアームを動かすとフォロワーアームが追従する状態を作ります（第7章）。テレオペレーションのイメージは次のコマンドです（詳細と正確な引数は第7章）。

```bash
conda activate lerobot
# リーダーで操作した動きに、フォロワーが追従する
python -m lerobot.teleoperate \
  --robot.type=so101_follower --robot.port=<フォロワーのポート> \
  --teleop.type=so101_leader --teleop.port=<リーダーのポート>
```

ここまでで、SO-101が「人間の手のコピー」として動くようになります。

## ステップ4：タスクを実演して記録する

覚えさせたいタスク（例：机の上のブロックを箱に入れる）を、リーダーアームで何十回も実演し、カメラ画像・関節角度・アクションを1エピソードずつ記録します（第8章）。

```bash
# タスクを繰り返し実演し、データセットとして記録・アップロード
python -m lerobot.record \
  --robot.type=so101_follower --robot.port=<ポート> \
  --teleop.type=so101_leader --teleop.port=<ポート> \
  --dataset.repo_id=<あなたのID>/pick_place --dataset.num_episodes=50
```

「何回・何ステップ集めればいいのか」というデータ設計の勘所は第9章・第10章で詳しく扱います。

## ステップ5：学習する

集めたデータでACTモデルを訓練します。学習はGPUが必要なので、Google Colab等のクラウドGPUを使うのが手軽です（第11章）。

```bash
python -m lerobot.scripts.train \
  --dataset.repo_id=<あなたのID>/pick_place \
  --policy.type=act --output_dir=outputs/train/pick_place
```

## ステップ6：ロボットが自分で動く

学習済みモデルを読み込ませると、ロボットが自律的にタスクを実行します。ここで成功率を測り、失敗を分析し、足りないデータを足す——この「評価と改善のループ」を回すのが上達の近道です（第13章）。

```bash
python -m lerobot.record \
  --robot.type=so101_follower --robot.port=<ポート> \
  --policy.path=outputs/train/pick_place/checkpoints/last \
  --dataset.repo_id=<あなたのID>/eval_pick_place --dataset.num_episodes=10
```

## この先の読み方

以上が、SO-101で模倣学習を1周させる全体像です。次章からは、この各ステップを実際に動かせるレベルまで掘り下げていきます。

**コラム：なぜSO-101は安いのか**

産業用ロボットが数百万円するのに対しSO-101が約7万円で実現できる理由は4つあります。①**精度要件の違い**——産業用は±0.05mmが必要ですが、学習ベースのSO-101は±1〜2mmで十分（誤差はデータで吸収できる）。②**量産効果**——STS3215サーボはホビー用途で大量生産され1個約4,000円と格安。③**オープンソースの力**——設計・R&Dコストがコミュニティで分散され、誰かの改良を全員が無償で使える。④**3Dプリント技術**——金属加工不要でPLAフィラメントでも十分な強度が得られる。研究・教育用途に特化することで、性能と価格の最適なバランスを実現しているのです。

# 第4章　組み立て手順 — リーダーアームとフォロワーアーム
---

## 4.1 組み立て前の準備

組み立ては清潔で広い作業スペース（60cm四方以上）で行います。所要時間は初めての場合で**リーダー約4〜5時間、フォロワー約4〜5時間、合計8〜10時間**が目安です。焦らず1つずつ確実に進め、間違えたら戻ることを恐れず（特にJoint 4〜5）、公式動画も参照しながら作業しましょう。

**部品の確認**：まず全部品が揃っているか確認します。3Dプリント部品にサポート材が付いている場合は、デザインナイフやラジオペンチで慎重に除去し、可動部の穴やネジ穴に残りがないか仕上げます。本体を傷つけないよう注意してください。

![画像](https://static.zenn.studio/user-upload/d44ffcda45ff-20251219.jpg)

電子部品（サーボ12個、制御基板2、3ピンケーブル12本以上、電源ケーブル2本、M2/M3ネジ）は小分けにして混ざらないよう整理します。

## 4.2 サーボモーターへのラベリング

SO-101のサーボは外見が同じでも内部のギア比が異なるため、事前にラベルを貼っておくと組み立て時のミスを防げます。

![画像](https://static.zenn.studio/user-upload/93cc4dbafa84-20251219.jpg)

| ラベル | ジョイント | ギア比（リーダー） | ギア比（フォロワー） | ID |
|---|---|---|---|---|
| 1 | Base / Shoulder Pan | 1:191 | 1:345 | 1 |
| 2 | Shoulder Lift | 1:345 | 1:345 | 2 |
| 3 | Elbow Flex | 1:191 | 1:345 | 3 |
| 4 | Wrist Flex | 1:147 | 1:345 | 4 |
| 5 | Wrist Roll | 1:147 | 1:345 | 5 |
| 6 | Handle（L）/ Gripper（F） | 1:147 | 1:345 | 6 |

リーダー側はL1〜L6、フォロワー側はF1〜F6とラベリングします。Pro版のフォロワー（F1〜F6）は12V仕様のモーターです。

## 4.3 リーダーアームの組み立て

各ジョイントに共通する基本の流れは、**①前のモーターとのケーブルを先に接続 → ②モーターを3Dプリント部品に挿入しM2ネジで固定 → ③モーターホーンを取り付けM3ホーン固定ネジで締める → ④次の構造部品をホーンにM3ネジ8本で固定**、です。

:::message
📷 **画像プレースホルダー**：ジョイント組み立ての基本4ステップ（ケーブル接続→モーター挿入→ホーン取り付け→構造部品固定）を1枚に図解したもの。各ジョイントで繰り返す共通パターンなので、最初に示すと全体が理解しやすい。
:::

**Joint 1：ベース（L1）** — L1モーターをBase_Leaderに挿入し（ケーブル出口は外側）、M2ネジで固定。Motor_Holder_1を被せ、ホーンを取り付けたら、Shoulder_LeaderをM3ネジ8本でしっかり固定します（ここが緩むとアーム全体がガタつきます）。Joint 1が滑らかに回転すればOK。

![画像](https://static.zenn.studio/user-upload/bcf7c0871a2b-20251219.png)

**Joint 2：肩（L2）** — **まずL1〜L2間の3ピンケーブルを接続**（後からでは困難）。L2モーターをShoulderに挿入（ケーブルはBase方向へ）、ホーンを付け、Upper_ArmをM3ネジ8本で左右対称に固定。アームが上下に動けばOK。

![画像](https://static.zenn.studio/user-upload/0b3edeb17d97-20251219.jpg)

**Joint 3：肘（L3）** — L2〜L3間のケーブルを接続し、L3モーターをUpper_Arm先端に固定。ホーンを付け、ForearmをM3ネジ8本で固定。肘が滑らかに曲がればOK。

![画像](https://static.zenn.studio/user-upload/6721fa238c73-20251222.jpeg)

**Joint 4：手首屈曲（L4）** — Motor_Holder_4をForearm先端に被せる際、**ケーブルの取り回しを確認**（挟み込み注意）。L3〜L4間のケーブルを接続してからL4モーターを挿入・固定し、ホーンを取り付けます。ここまでで全ケーブルが正しく接続されているか確認します。

![画像](https://static.zenn.studio/user-upload/c130d5f1caab-20251222.jpeg)

**Joint 5：手首回転（L5）⚠️最重要** — 多くの人がここで間違えてやり直しになります。**順序が肝心です**：

```
✅ 正解： 先にL5をWrist_Holderに挿入（前面M2ネジ2本のみ）→ ケーブル接続 →
         上側だけホーン取り付け → Wrist_Holder全体をJoint 4にM3ネジ8本で固定
❌ 誤り： 先にWrist_HolderをJoint 4に固定 → L5を入れようとして入らない／締められない
```

下側にはホーンを付けません（Wrist_Holderと干渉します）。手首が屈曲・回転の2軸で動けばOK。

![画像](https://static.zenn.studio/user-upload/ebe0fe4c01bd-20251222.jpeg)

**Joint 6：ハンドル（L6）** — L5〜L6間のケーブルを接続し、HandleをL5モーターのホーンにM3ネジ4本で固定。リーダーのハンドル部モーターは、人間が握って動かすため実際には使わないことも多いです。

![画像](https://static.zenn.studio/user-upload/be72f166f295-20251222.jpeg)

## 4.4 フォロワーアームの組み立て

フォロワーアームはリーダーとほぼ同じ手順ですが、①全モーターがF1〜F6（ギア比1:345統一、Pro版は12V）、②Base/Shoulderがフォロワー専用部品、③先端がハンドルではなく**グリッパー**、という違いがあります。Joint 1〜5はリーダーと同じ手順（ケーブル接続のタイミング、Joint 5の順序も同様）で組みます。

![画像](https://static.zenn.studio/user-upload/47837d5dbfd0-20251222.jpeg)

**Joint 6：グリッパー（F6）** — フォロワー最大の違いです。Gripper_BaseをF5モーターのホーンにM3ネジ4本で固定し、F5〜F6間のケーブルを接続してF6モーターをGripper_Baseに挿入・固定。ホーンを取り付けたら、Gripper_ClawをM3ネジ8本で固定します。グリッパーが開閉すればOK。

![画像](https://static.zenn.studio/user-upload/37cb81db493b-20251222.jpeg)

## 4.5 制御基板の取り付け

両アームが組み上がったら制御基板を取り付けます。リーダーはBase_Leader、フォロワーはBase_Followerの内部または側面に配置し、M3ネジまたはスペーサーで固定します（付属の3Dプリント固定具は穴径が大きい場合があるため、金属ナット・ワッシャーの併用を推奨）。各アームのJoint 1モーターからのケーブルを基板に接続し、USB・電源コネクターが外部からアクセスできることを確認します。**Pro版は12V電源を接続し、極性を間違えないよう注意**してください。ケーブルは結束バンドやスパイラルチューブでまとめ、可動部には余裕を持たせます。

## 4.6 クランプでの固定

各アームを2個ずつのクランプで机に固定します。リーダーとフォロワーは向かい合わせ、または並列に配置します。しっかり締めつつ机を傷めないよう、ゴムパッドやフェルトを挟むとよいでしょう。動作中にズレないことを確認します。

:::message
📷 **画像プレースホルダー**：リーダーアームとフォロワーアームを机にクランプ固定し、カメラも配置した「作業レイアウトの完成図」。俯瞰カメラと手首カメラの位置関係が分かる構図。次章以降のキャリブレーション・テレオペの基準になる。
:::

## 4.7 まとめ

本章では組み立て手順を解説しました。要点は、**サーボのラベリング**（ギア比を間違えない）、**ケーブルは組み立て前に接続**、**Joint 5は先にモーターをWrist_Holderに挿入**、そして各段階での動作確認です。次章では、PCからアームを制御するためのソフトウェア環境を構築します。

---

**コラム：組み立ての学び**

SO-101の組み立ては、ロボットの構造やメカニカルデザインの基本を体験的に学ぶ機会です。特に「なぜこの順序なのか」を考えることが重要——Joint 5で先にモーターを挿入する理由、ケーブルを事前接続する理由は、いずれも「組み立て性（Assemblability）」の原則です。組み立て中に感じた「ここはこうすれば」という気づきは、オープンソースへの貢献の種になります。実際、Seeed公式ブログでは日本のユーザーが改良した固定パーツがプルリクエストとして採用されました。あなたのアイデアも次のバージョンに反映されるかもしれません。

# 第5章　ソフトウェア環境のセットアップ（macOS + Conda）
---

## 5.1 環境構築の全体像

SO-101を動かすには、MacにLeRobotライブラリと関連ソフトを入れる必要があります。本章では **Conda環境マネージャ** を使い、macOS環境の確認 → Miniforge（Conda）→ Python 3.10専用環境 → LeRobot本体＋Feetech SDK → PyTorch → 動作確認、の順に構築します。所要時間は1〜2時間が目安です。

Condaを使う理由は、Python本体とパッケージをプロジェクトごとに完全分離でき（システムのPythonを汚さない）、`environment.yaml`でエクスポートして再現でき、Apple Silicon（M1/M2/M3）にネイティブ対応しているためです。

:::message
📷 **画像プレースホルダー**：環境構築の全体フロー図。「Homebrew → Miniforge(Conda) → lerobot環境 → LeRobot+Feetech SDK → PyTorch → カメラ/HF設定 → 動作確認」の依存関係を上から下への流れで示す。詰まったときに「今どこか」を確認できる図。
:::

## 5.2 macOS環境の確認

推奨は macOS 12 (Monterey) 以降です。バージョンとアーキテクチャを確認しておきます。

```bash
sw_vers        # macOSバージョン
uname -m       # arm64 = Apple Silicon / x86_64 = Intel Mac
```

macでできるのは**データ収集・テレオペ・データセット作成・可視化・推論（CPU/MPS）**まで。**GPU（CUDA）を使う本格的な学習はできない**ため、学習はGoogle Colab等のクラウドGPUに任せるのが標準的な構成です。

## 5.3 Homebrew と Miniforge のインストール

パッケージマネージャの Homebrew を入れます（未インストールの場合）。

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
# Apple Silicon の場合、パスを通す
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

続いて Conda（Miniforge）を入れます。Miniforgeは軽量でconda-forgeがデフォルト、Apple Siliconに完全対応です。

```bash
brew install miniforge
conda init "$(basename "${SHELL}")"
# ターミナルを再起動後、動作確認
conda --version
```

プロンプトに `(base)` が表示されればOKです。チャンネル設定をしておきます。

# 第6章　モーター設定とキャリブレーション
---

## 6.1 なぜキャリブレーションが必要か

組み立てたSO-101は、そのままでは使えません。各モーターに固有のIDを割り当てる**モーター設定**と、リーダーとフォロワーの位置を対応付ける**キャリブレーション**が必要です。

キャリブレーションはSO-101で最も重要なセットアップ作業です。リーダーとフォロワーが同じ角度値で同じ姿勢を取れるようにすることでテレオペが正しく動き、異なるSO-101間でもデータやモデルを共有できる再現性が生まれ、模倣学習の精度も左右します。初回の所要時間は2〜3時間が目安です。

## 6.2 作業の全体フロー

```
1. USBポートの特定 → 2. ファームウェア更新（Windows推奨）→ 3. モーターID設定（1台ずつ）
→ 4. フォロワーのキャリブレーション → 5. リーダーのキャリブレーション → 6. 保存と検証
```

**重要**：モーターIDの設定は必ず1台ずつ行い、ファームウェア更新にはWindowsが必要です。

:::message
📷 **画像プレースホルダー**：モーター設定〜キャリブレーションの6ステップを1枚にまとめたフローチャート。各ステップに使うツール名（lerobot-find-port / FD1.9.8.5 / lerobot-setup-motors / lerobot-calibrate）を添える。
:::

## 6.3 USBポートの特定

リーダーとフォロワーの2つの制御基板がそれぞれどのポートかを特定します。両方を接続した状態で `lerobot-find-port` を実行し、片方のケーブルを抜いてどのポートが消えるかで判別します。

```bash
lerobot-find-port
# → ['/dev/tty.usbmodem575E0032081', '/dev/tty.usbmodem575E0031751']
# 一方を抜いて Enter → 消えたポートがそのアーム
```

判別したポート名（リーダー／フォロワー）を記録します。ポート名は環境により異なります。

## 6.4 ファームウェアの更新（Windows必須）

Feetech STS3215は出荷時のファームウェアが古い場合があり、通信エラーや位置制御精度の低下、キャリブレーション失敗の原因になります。**更新はWindowsでのみ可能**です（Feetech公式ツール **FD1.9.8.5**、[Gitee](https://gitee.com/ftservo/fddebug) からダウンロード）。

手順は、モーターを**1台だけ**制御基板に接続し、FD1.9.8.5で「Scan」→検出したモーターを選択→「Firmware」タブで最新の`.bin`を選び「Update」、というものです。**更新中は絶対に電源やUSBを抜かないでください**（モーター破損の恐れ）。これを12台すべてに繰り返します（合計15〜30分）。

## 6.5 サーボモーターIDの設定

各モーターに1〜6のIDを割り当てます。リーダー・フォロワーとも、Base=1、Shoulder=2、Elbow=3、Wrist Flex=4、Wrist Roll=5、Handle/Gripper=6です（別々の制御基板なので同じ番号でOK）。

:::message
📷 **画像プレースホルダー**：SO-101のアーム図に各ジョイントとID番号（1〜6）を吹き出しで示した「ID割り当て図」。どのモーターがどのIDかを組み立て時のラベルと対応させて確認できる図。
:::

LeRobotの自動設定ツールが手軽です。フォロワーの制御基板のみを接続し、次を実行します。

```bash
lerobot-setup-motors --robot.type=so101_follower --robot.port=<フォロワーのポート>
```

プロンプトに従い、**該当するモーターだけを接続してEnter**を繰り返します（Gripper=6 → Wrist Roll=5 → … → Base=1 の順）。毎回、指定されたモーターのみが接続されていることを確認してください。リーダーも同様に `--teleop.type=so101_leader --teleop.port=<リーダーのポート>` で設定します。ボーレートは自動で1000000（1Mbps）に設定されます。

## 6.6 キャリブレーションの準備

キャリブレーションは、**物理的な角度とモーター内部の数値を対応付ける**作業です。たとえば「肩が水平＝モーター値2048、90度上げた状態＝3072」という対応を記録します。作業は**①ゼロ位置（各ジョイントを可動範囲の中央）の設定**と**②各ジョイントの可動範囲の記録**の2段階です。

キャリブレーション中は電源ON・トルクOFF（脱力）で、アームを手で動かします。急な動きは避けてゆっくり動かし、ケーブルが引っかかっていないか常に確認してください。アームはクランプでしっかり固定しておきます。

## 6.7 フォロワーアームのキャリブレーション

```bash
lerobot-calibrate --robot.type=so101_follower --robot.port=<ポート> --robot.id=my_follower_arm
```

まず全ジョイントを可動範囲の中央（Base=正面、Shoulder=水平、Elbow=約90度、Wrist=真っ直ぐ、Gripper=半開き）にして**ゼロ位置**を記録します（完璧な中央でなく「だいたい中央」で大丈夫）。次に、各ジョイントを最小位置→Enter、最大位置→Enter、と6ジョイント分繰り返します。完了すると `~/.cache/lerobot/calibration/my_follower_arm.json` に保存されます。

:::message
📷 **画像プレースホルダー**：キャリブレーションの「ゼロ位置」姿勢の写真または図。各ジョイントを可動範囲の中央に置いたSO-101の全体像を、Base/Shoulder/Elbow/Wrist/Gripperのラベル付きで示す。読者が正しいゼロ位置を再現できる図。
:::

## 6.8 リーダーアームのキャリブレーション

リーダーも同じ手順ですが、パラメータ名が `--teleop` になる点に注意します。

```bash
lerobot-calibrate --teleop.type=so101_leader --teleop.port=<ポート> --teleop.id=my_leader_arm
```

リーダーはギア比が軽く手で動かしやすい反面、可動範囲が広くなります。リーダーは「人間が操作しやすい範囲」に設定すればよく、フォロワーより少し狭めでも問題ありません。

## 6.9 キャリブレーションの検証

最も確実な確認は、実際にテレオペを試すことです（詳細は次章）。

```bash
lerobot-teleoperate \
  --teleop.type=so101_leader --teleop.port=<リーダーのポート> \
  --robot.type=so101_follower --robot.port=<フォロワーのポート>
```

リーダーを動かすとフォロワーが同じ姿勢で追従し、滑らかに動けば成功です（多少のズレは許容範囲）。姿勢が大きくずれる場合はキャリブレーションをやり直します。

## 6.10 データの管理と再キャリブレーション

キャリブレーションデータは `~/.cache/lerobot/calibration/` に保存されます。**必ずバックアップ**しておきましょう。

```bash
mkdir -p ~/lerobot_backups/calibration
cp ~/.cache/lerobot/calibration/*.json ~/lerobot_backups/calibration/
```

アームを分解・再組み立てした、モーターを交換した、テレオペで位置がずれる、長期間使っていなかった——こうした場合は再キャリブレーションが必要です。ずれているジョイントだけの再調整も可能ですが、全体をやり直す方が確実です。定期メンテナンス（3ヶ月に1回程度）も推奨します。

## 6.11 トラブルシューティング

| 症状 | 主な原因と対処 |
|---|---|
| モーターが検出されない | ケーブル・電源・ポート名を確認、別のUSBポートを試す |
| ID設定ができない | モーターは必ず1台ずつ接続、ファームウェアを更新、ボーレート（1000000）を確認 |
| キャリブレーション中にエラー | ツール再起動、ゆっくり動かす、ケーブルを整理 |
| リーダーとフォロワーがずれる | ゼロ位置を慎重に再設定、可動範囲を記録し直す、モーターのラベルを再確認 |
| ファームウェア更新が失敗 | モーター再接続、ファイル再ダウンロード、安定した電源を使用 |

## 6.12 まとめ

本章では、USBポートの特定、ファームウェア更新、モーターID設定、リーダー・フォロワーのキャリブレーションまでを完了しました。要点は、**IDは1台ずつ**、**ファームウェアは最新版に**、**キャリブレーションは丁寧に**、そして**データは必ずバックアップ**することです。次章では、いよいよテレオペレーションを実践します。

---

**コラム：キャリブレーションの数学**

キャリブレーションで記録している数値は、実は**線形変換のパラメータ**です。サーボの内部値（0〜4095）を角度（ラジアン）に変換する式は `angle_rad = (motor_value - zero) × scale` で、`zero` はゼロ位置のモーター値、`scale` は可動範囲から計算されるスケールファクターです。たとえば可動範囲が min=1024, max=3072 なら `scale ≈ (max−min)/(2π) ≈ 326 counts/radian`。このシンプルな線形変換のおかげで、異なるロボット間でも同じ「角度」を共有でき、模倣学習の転移可能性の基礎になっています。より高度なキャリブレーションでは非線形補正やバックラッシュ補償も扱いますが、SO-101では線形近似で十分な精度が得られます。

# 第7章　テレオペレーションの基礎
---

## 7.1 テレオペレーションとは

キャリブレーションが完了したSO-101で、いよいよ**テレオペレーション（遠隔操作）**を体験します。テレオペレーションとは、**人間がリーダーアームを動かすと、フォロワーアームがリアルタイムで同じ動きを追従する**仕組みです。

```
人間の手 → リーダーアーム（入力）→ 制御プログラム → フォロワーアーム（出力）
```

これにより、人間の動きを直接ロボットに伝え、複雑な動作もプログラミングなしで実行でき、そして何より**模倣学習のためのデータ収集**が可能になります。テレオペの真の目的は「遠隔操作」そのものではなく、次章以降で使う「デモンストレーション（お手本）の記録」にあります。

## 7.2 テレオペレーションの仕組み

テレオペは、次のループを高速（通常30Hz＝1秒に30回）で繰り返しています。

```python
while True:
    leader_position = leader.read()       # リーダーの現在位置を読む
    follower.write(leader_position)        # フォロワーへ送る
    sleep(0.033)                           # 30Hz
```

第6章で作ったキャリブレーションデータが、「リーダーのモーター値→角度→フォロワーのモーター値」という変換を担い、両者が同じ姿勢を取れるようにします。完全なリアルタイムではなく、通信遅延・モーター応答・処理時間を合わせて約30〜50msの遅延が生じるのが一般的です。

:::message
📷 **画像プレースホルダー**：テレオペの制御ループ図。「リーダー位置を読む → キャリブレーションで変換 → フォロワーに書く → 30Hzで反復」の循環を、リーダー/フォロワーのイラストと数値変換（モーター値↔角度）とともに示す。
:::

## 7.3 テレオペレーションの実行

両アームを机に固定し、制御基板をUSB・電源に接続したら、次のコマンドで起動します（ポート名は第6章で記録したもの）。

```bash
lerobot-teleoperate \
  --robot.type=so101_follower --robot.port=<フォロワーのポート> \
  --teleop.type=so101_leader --teleop.port=<リーダーのポート> \
  --fps=30
```

起動すると接続確認とウォームアップが行われます。**ウォームアップ中はアームに触れないでください**。「Teleoperation started」と表示されたら、リーダーアームを動かすとフォロワーが追従します。停止は `Ctrl+C`、緊急時は電源を切ればモーターが脱力します。

**動作確認**として、①Joint 1（ベース）だけをゆっくり回してフォロワーが追従するか、②アーム全体を大きく動かして全ジョイントが滑らかに追従するか、③リーダーのハンドルを握るとフォロワーのグリッパーが閉じ、離すと開くか、を順に確かめます。

## 7.4 動作の確認と調整

**同期精度**：リーダーを特定の姿勢にしたとき、フォロワーが同じ姿勢になっているか目視で確認します。角度差±5度程度は許容範囲で、明らかにずれる場合はキャリブレーションを見直します。

**遅延とFPS**：リーダーを素早く動かして追従の速さを見ます。正常値の目安はLatency 10〜30ms、FPS 28〜32（30Hz設定時）です。Latencyが50msを超える、FPSが大きく落ちる場合は通信や電源に問題がある可能性があります。フォロワーが小刻みに震える場合はPIDゲイン、脱力する場合は電源供給を疑います。

## 7.5 モーター制御パラメータ

サーボモーターには**トルク**（出力する力）を設定できます。重い物体を持てないならトルクを上げ、モーターが熱くなるなら下げます。**速度制限**は、急な動きを防いで安全性やデータ収集の安定性を高めるのに有効です。位置制御には**PID制御**が使われており、Pゲインを上げると素早く動く一方オーバーシュートしやすく、Dゲインを上げると振動を抑えられます。通常は調整不要ですが、振動が激しい場合に検討します。

## 7.6 Rerun Viewerでの可視化

LeRobotは、テレオペの状態をリアルタイムに可視化できます。起動コマンドに `--display=true` を付けると Rerun Viewer が立ち上がり、アームの3Dビュー、各ジョイント角度のグラフ（リーダー vs フォロワーの比較）、FPS・Latency・エラー率などの統計が表示されます。どのジョイントがずれているか、動きが滑らかか、を視覚的にデバッグできます。

:::message
📷 **画像プレースホルダー**：Rerun Viewer の画面キャプチャ。左に3Dアームモデル、右にリーダー/フォロワーのジョイント角度グラフとFPS/Latency表示が並んだ様子。テレオペが正常に動いているときの「良い状態」の見本。
:::

## 7.7 記録とリプレイ — データ収集への橋渡し

テレオペ中の動きは、記録して後で再生できます。この「記録」こそが、次章の模倣学習用データセット作成の基礎です。

```python
# 記録：リーダーの軌跡を配列に貯める
trajectory = []
while running:
    pos = leader.read()
    trajectory.append(pos)
    follower.write(pos)

# リプレイ：貯めた軌跡をフォロワーで再生
for pos in trajectory:
    follower.write(pos)
    time.sleep(1/30)
```

人間がリーダーアームで作業を実演し、その軌跡（＋カメラ画像）を記録する——これが模倣学習の第一歩です。次章では、この記録機能を使って実際にデータセットを作成します。

## 7.8 まとめ

本章では、テレオペレーションを起動し、リーダーとフォロワーの同期・遅延・滑らかさを確認しました。要点は、**テレオペは30Hzのループで動く**こと、**キャリブレーションが両者を同期させる**こと、**遅延30〜50msは正常**であること、そして**テレオペの本質はデータ収集の準備**であることです。次章では、カメラを含めた撮影環境でデモを記録し、模倣学習用データセットを作ります。

**コラム：テレオペレーションの歴史**

テレオペレーションの概念は古く、1940年代の放射性物質を扱う遠隔マニピュレーター（マスター・スレーブ方式）、1960年代の月面探査ロボット、1990年代の外科手術ロボット「da Vinci」へと発展してきました。SO-101が使うのは最もシンプルな「位置ベースのマスター・スレーブ制御」——リーダーの位置をそのままフォロワーに送るだけです。複雑な力覚フィードバックやバイラテラル制御は不要。なぜなら、SO-101の目的は「遠隔操作」ではなく**「データ収集」**だからです。人間が作業を実演し、その軌跡を記録する。これが模倣学習の出発点になります。
